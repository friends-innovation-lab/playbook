## Why the Treehouse exists

The federal contracting landscape is changing. Agencies want to see working prototypes, not slide decks. Teams that can show up with something real and functional — fast — win more work. Beyond proposals, this is also where Friends builds internal tools, product ideas, and experiments. The Treehouse makes it possible to move at that speed while keeping the work production-ready.

---

## What's distinctive about it

The lab moves fast because the infrastructure is real and pre-built. CI, accessibility, security, performance — all wired in before the first line of project code gets written. When something feels easy, it's because the work is done. The Treehouse handles the engineering rigor so the creative work gets the attention.

## A few things to know

- **Asking is encouraged.** This is the first version of the Treehouse. Some things will be unclear. Better to ask ten times than struggle silently.
- **This isn't only for developers.** The infrastructure handles the technical setup. Claude.ai handles the planning. Claude Code handles the implementation. Anyone who can describe what they want can build here.
- **This is meant to feel possible.** If at any point it feels like too much, that's a docs problem, not a Friends problem. Surface it. It'll get fixed.
