## Code Standards

### Critical Rules
1. **No server-side redirects on landing pages** — use next.config.js redirects instead
2. **Type everything** — no `any` types without a comment explaining why
3. **Error states are required** — every data fetch needs loading, error, and empty states
4. **Mobile-first** — all layouts must work on 375px width
5. **RLS on every table** — no exceptions, ever
6. **Never use service role key in client components**

### New Packages
Before installing anything new, check if the functionality already
exists in the current stack. Prefer packages already used here.

### Environment Variables
Never hardcode values that should be env vars.
All new env vars must be added to `.env.example` with a description.
