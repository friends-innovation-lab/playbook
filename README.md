# Friends Innovation Lab - Playbook

The operational manual for running the Innovation Lab. This playbook contains everything our team needs: methodology for working with clients, operational processes, and technical standards for building software.

**Who is this for?** Friends Innovation Lab team members — Project Leads, UX Strategists, Designers, and Builders.

## Structure

```
playbook/
├── methodology/              # PIM - client-facing methodology
│   ├── overview.md           # Public Impact Method documentation
│   ├── ground/               # Phase 1: Legitimacy & constraints

│   ├── shape/                # Phase 3: Viable options
│   ├── test/                 # Phase 4: Real proof
│   └── embed/                # Phase 5: Durable change
│
├── operations/               # Internal - how we run the lab
│   ├── first-time-setup.md   # Tool installation, one-time setup

│   ├── automation/           # Scripts (spinup.sh)
│   └── design/               # UI/UX standards, components
│
├── development/              # Technical standards
│   ├── frontend/             # React, Next.js, styling, accessibility
│   ├── backend/              # Database, auth, APIs
```
